Bean 后处理器 trait Bean post-processor trait

在 Bean 实例化之后对 Bean 进行加工，支持初始化前和初始化后两个扩展点。 Processes beans after instantiation, with before-init and after-init hooks.

等价于 Spring 的 BeanPostProcessor。 Equivalent to Spring’s BeanPostProcessor.

§Lifecycle Position / 生命周期位置

Instantiation -> before_init -> @PostConstruct -> after_init -> Ready

§示例 / Example

ⓘ
use hiver_starter::core::bean_post_processor::{BeanPostProcessor, BeanContext};

struct MyPostProcessor;

impl BeanPostProcessor for MyPostProcessor {
    fn post_process_before_initialization(
        &self,
        bean: &mut dyn Any,
        context: &BeanContext,
    ) -> anyhow::Result<()> {
        // 初始化前的处理
        // Processing before initialization
        Ok(())
    }

    fn post_process_after_initialization(
        &self,
        bean: &mut dyn Any,
        context: &BeanContext,
    ) -> anyhow::Result<()> {
        // 初始化后的处理
        // Processing after initialization
        Ok(())
    }
}
