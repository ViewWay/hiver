1//! Hiver Runtime - async runtime on async-executor / async-io
2//! Hiver 运行时 - 基于 async-executor / async-io 的异步运行时
3//!
4//! # Overview / 概述
5//!
6//! `hiver-runtime` builds on the [`async-executor`] + [`async-io`] + [`async-net`]
7//! ecosystem (the same crates `smol` is composed of): a multi-task executor driven
8//! by a reactor-aware `block_on`, with cross-platform epoll/kqueue/IOCP I/O.
9//! This keeps the public surface stable (`Runtime`, `block_on`, `spawn`,
10//! `io::*`, `time::*`) while delegating the low-level scheduling and I/O to
11//! battle-tested libraries.
12//!
13//! `hiver-runtime` 构建于 [`async-executor`] + [`async-io`] + [`async-net`] 生态
14//! （与 `smol` 由相同 crate 组成）:一个由 reactor 感知 `block_on` 驱动的多任务
15//! 执行器,提供跨平台 epoll/kqueue/IOCP I/O。这保持了公共接口稳定
16//! （`Runtime`、`block_on`、`spawn`、`io::*`、`time::*`）,同时将底层调度与 I/O
17//! 委托给久经验证的库。
18//!
19//! # Features / 功能
20//!
21//! - Multi-task executor via [`async_executor::Executor`] / 经 [`async_executor::Executor`]
22//!   的多任务执行器
23//! - Reactor-aware `block_on` (smol's driver) / reactor 感知的 `block_on`（smol 的驱动器）
24//! - Cross-platform async I/O (TCP/UDP) via async-net / 经 async-net 的跨平台异步 I/O（TCP/UDP）
25//! - Timers via async-io / 经 async-io 的定时器
26//! - Fire-and-forget `spawn` (task is detached on handle drop) / fire-and-forget
27//!   `spawn`（句柄丢弃时 detach 任务）
28//!
29//! # Example / 示例
30//!
31//! ```rust,no_run,ignore
32//! use hiver_runtime::Runtime;
33//!
34//! fn main() -> std::io::Result<()> {
35//!     let runtime = Runtime::new()?;
36//!     runtime.block_on(async {
37//!         println!("Hello, Hiver!");
38//!     });
39//!     Ok(())
40//! }
41//! ```
