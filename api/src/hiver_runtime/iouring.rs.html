2//! io_uring reactor еҗҺз«Ҝ(д»… Linux,feature `io-uring`)гҖӮ

4//! **Status: reserved / not yet implemented.**
5//! **зҠ¶жҖҒ:йў„з•ҷ / е°ҡжңӘе®һзҺ°гҖӮ**
6//!
7//! This module is the integration point for a future high-performance
8//! thread-per-core io_uring reactor, modeled after [monoio]. When the
9//! `io-uring` feature is active on Linux, `Runtime::with_config` should prefer
10//! an [`IoUringDriver`] over the default `async-io` reactor.
11//!
12//! жӯӨжЁЎеқ—жҳҜжңӘжқҘй«ҳжҖ§иғҪ thread-per-core io_uring reactor зҡ„жҺҘе…ҘзӮ№,еҸӮиҖғ [monoio]
13//! е»әжЁЎгҖӮеҪ“ `io-uring` feature еңЁ Linux дёҠжҝҖжҙ»ж—¶,`Runtime::with_config` еә”дјҳе…Ҳ
14//! йҖүз”Ё [`IoUringDriver`] иҖҢйқһй»ҳи®Өзҡ„ `async-io` reactorгҖӮ
15//!
16//! # Why io_uring? / дёәдҪ•з”Ё io_uring?
17//!
18//! io_uring delivers true zero-syscall-storm async I/O: submissions and
19//! completions are batched in shared ring buffers, eliminating the per-call
20//! `epoll_wait`/`read`/`write` syscalls. Combined with thread-per-core (no
21//! cross-thread work stealing), this is the lowest-latency async model on
22//! Linux. See <https://kernel.dk/io_uring.pdf> and the monoio architecture.
23//!
24//! io_uring жҸҗдҫӣзңҹжӯЈзҡ„йӣ¶зі»з»ҹи°ғз”ЁйЈҺжҡҙејӮжӯҘ I/O:жҸҗдәӨдёҺе®ҢжҲҗеңЁе…ұдә«зҺҜеҪўзј“еҶІеҢәдёӯ
25//! жү№еӨ„зҗҶ,ж¶ҲйҷӨжҜҸж¬Ўи°ғз”Ёзҡ„ `epoll_wait`/`read`/`write` зі»з»ҹи°ғз”ЁгҖӮй…ҚеҗҲ
26//! thread-per-coreпјҲж— и·ЁзәҝзЁӢ work stealing),иҝҷжҳҜ Linux дёҠжңҖдҪҺе»¶иҝҹзҡ„ејӮжӯҘжЁЎеһӢгҖӮ
27//! и§Ғ <https://kernel.dk/io_uring.pdf> еҸҠ monoio жһ¶жһ„гҖӮ
28//!
29//! # Implementation plan / е®һзҺ°и®ЎеҲ’
30//!
31//! 1. Add `io-uring = "0.7"` under `[target.'cfg(target_os = "linux")'.dependencies]` (gated by the
32//!    `io-uring` feature) in `Cargo.toml`.
33//! 2. Implement `IoUringDriver` here: a `Driver`-equivalent that owns an `io_uring::IoUring`
34//!    instance, registers FDs as `SubmissionQueuee`s, and drains the completion queue into wakers вҖ”
35//!    mirroring `async-io`'s `Reactor::react()` but with ring buffers instead of epoll.
36//! 3. In `runtime.rs`, add `#[cfg(all(target_os="linux", feature="io-uring"))]` selection logic in
37//!    `Runtime::with_config` to build an `IoUringDriver` instead of relying on `async-io`.
38//! 4. The `async-net` I/O types would need io_uring-aware equivalents (or use `monoio`'s net
39//!    types), since async-net is bound to the async-io reactor.
40//!
41//! # Why this is deferred / дёәдҪ•жҡӮзј“
42//!
43//! This requires a Linux environment to build, run, and benchmark. On macOS
44//! (the current dev platform) io_uring does not exist, so the implementation
45//! cannot be validated here. The reservation (feature flag + this module) lets
46//! a Linux contributor pick up the work with a clear integration point.
47//!
48//! иҝҷйңҖиҰҒ Linux зҺҜеўғжқҘжһ„е»әгҖҒиҝҗиЎҢдёҺеҹәеҮҶжөӢиҜ•гҖӮеңЁ macOSпјҲеҪ“еүҚејҖеҸ‘е№іеҸ°пјүдёҠ
49//! io_uring дёҚеӯҳеңЁ,ж•…е®һзҺ°ж— жі•еңЁжӯӨйӘҢиҜҒгҖӮжӯӨйў„з•ҷпјҲfeature flag + жң¬жЁЎеқ—пјүдҪҝ
50//! Linux иҙЎзҢ®иҖ…еҸҜеңЁжҳҺзЎ®зҡ„жҺҘе…ҘзӮ№з»§з»ӯиҜҘйЎ№е·ҘдҪңгҖӮ
51//!
52//! [monoio]: https://github.com/bytedance/monoio
